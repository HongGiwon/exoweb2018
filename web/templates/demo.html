    2) 정성 평가: 난제 유형의 문제에 대한 정확도/정밀도 비교 (고난도 문제 해결능력 측정)


    ◆ CGQA Architecture Ver. 0 (프로토타입)의 구성

    • 본 Ver. 0는 end-to-end 파이프라인 구조로서, QA에 대해 자동화되어 구동됨.

    영문 위키피디아 문서를 이용해 학습한 개념들의 임베딩*

    *임베딩 : 문서를 구성하는 하나의 단어를 수치화하는 방법의 일종

    ◆ 데이터 정보

     구축 건수

     32,651,644 개념

     데이터 구조

      -개념(단어)
    -임베딩(50차원)


    ◆ 데이터 설명

    □ 개요
      o 문서 속의 맥락을 인지하기 위하여, 문서 내에 등장하는 개별 개념 단어들을 50차원의 벡터로 임베딩하는 학습 모델을시킨 결과물

    □ 세부내용
      o 임베딩 목표로 하는 개념을 정의하는 문서가 있는 경우에는 임베딩 방법론으로 Paragraph Vector 방법을 사용
      o 임베딩 목표로 하는 개념을 정의하는 문서가 따로 없는 경우에는, 목표 개념이 등장하는 문서들을 활용함
        - 문서내의 모든 단어를 임베딩하는 방법론인 word2vec를 활용하여 문서 내의 모든 단어를 벡터로 수치화한 후 목표 개념 단어의 벡터를 개념 임베딩으로 사용


    ◆ 구축 방법

    o 개념 정의 문서가 존재하는 경우
      - Paragraph Vector 방법을 적용하여 개념 정의 문서의 임베딩을 학습한 뒤 개념의 임베딩으로 활용
    o개념 정의 문서가 존재하지 않는 경우
      - 전체 영문 위키피디아 말뭉치를 이용한 Word2Vec 방법으로 개별 개념의 임베딩을 학습


    영문 위키피디아 문서를 이용해 학습한 컨텍스트들의 임베딩*

    *임베딩 : 문서를 구성하는 하나의 단어를 수치화하는 방법의 일종

    ◆ 데이터 정보

     구축 건수

     위키피디아 문서 10K+에 대한 컨텍스트 개체 (시간=190만 건, 장소=339만 건) 및 토픽정보 임베딩

     데이터 구조

      -문서(문서ID)
    -임베딩(300차원)


    ◆ 데이터 설명

    □ 개요
      o 행 별로 문서ID와 각 셀에 실수값을 가진 300차원의 벡터가 탭 문자(\t)로 구분되어 나열되어있음

    □ 세부내용
      o 위키피디아 본문 말뭉치에서 컨텍스트 개체 기반 문서 surrogate 생성 후 단어 임베딩 기술 적용
        - 시간: TimeML 기준 규칙 기반
        - 장소: 세계 지명 사전(GeoNames) 활용한 추출 (9백만 개 이상의 지명 및 대용어 포함)
        - 토픽: 문서 별 위키피디아 주제 카테고리 이용 및 지도학습 기반 질문의 토픽 분류
      o 비계층형 컨텍스트 연관관계 통합 학습의 효과
        - 모든 유형의 컨텍스트 간 연관관계 계산 가능한 임베딩 벡터 생성
        - 암시된 컨텍스트 매칭 가능 (ex. ‘미국’+ ‘1991년’ ≈ ‘걸프 만’)


    ◆ 구축 방법

    o 컨텍스트 추출 및 저장 질문 및 문서에 명시된 컨텍스트 추출
      - 위키피디아 본문 말뭉치에서 컨텍스트 개체 기반 문서 surrogate 생성 후 단어 임베딩 기술 적용
      - 절, 문장, 단락 단위에서 추출된 컨텍스트의 통합 저장

    형태소-동형이의어 주석 말뭉치*

    * 말뭉치: 언어 분석 연구를 위해 텍스트를 컴퓨터가 읽을 수 있는 형태로 주석을 추가한 자료로, 기계학습을 위한 학습데이터로 사용됨

    ◆ 데이터 정보

     구축 건수

     16,000,000어절

     데이터 구조

      -원문
    -태깅된 문장


    ◆ 데이터 설명

    □ 개요
      o 표준국어대사전의 어깨번호를 기준으로 동형이의어 대상 어깨번호 부착 말뭉치

    □ 세부내용
      o 세종형태의미말뭉치의 여러 오류(형태소분석,어깨번호)를 수정하고, 세종형태의미말뭉치에 미부착된 동형이의어(실질형태소, 접사)를 대상으로 표준국어대사전 기준의 어깨번호 부착
      o 표준국어대사전 뜻풀이/용례를 대상으로 형태의미주석
      o 조선일보, 경향신문 2015.04~2016.12년 형태의미주석
      o 법률(노동법, 민법, 상법시행법, 헌법, 형법)
      o 초등학교교과서(4학년 5학년 읽기/말하기/듣기/쓰기)


    ◆ 구축 방법

    o UTagger를 이용하여 형태의미주석 후 오류 수작업으로 수정

    형태소-동형이의어-의존관계-의미역 주석 말뭉치*

    * 말뭉치: 언어 분석 연구를 위해 텍스트를 컴퓨터가 읽을 수 있는 형태로 주석을 추가한 자료로, 기계학습을 위한 학습데이터로 사용됨

    ◆ 데이터 정보

     구축 건수

     2,000,000어절

     데이터 구조

      -원문
    -태깅된 문장


    ◆ 데이터 설명

    □ 개요
      o 약 2,000,000 어절의 세종말뭉치를 대상으로 형태소분석, 의미(동형이의어)분석, 의존관계분석, 의미역분석 말뭉치를 구축

    □ 세부내용
      o 31개의 구구조말뭉치는 의존관계로 변경
        - 어깨번호가 부착되지 않은 10개의 구구조말뭉치는 모두 어깨번호 부착
        - 세종형태의미말뭉치에 포함된 21개 파일

      o 39의 추가 세종형태의미말뭉치를 의존관계 및 의미역 부착

      o 의미역은 세종전자사전의 의미역 16개에 6개의 의미역을 추가하여 22개의 의미역
        - 행동주(AGT), 경험주(EXP), 피동주(PAT), 동반자(COM), 대상(THM), 기점(SRC), 착점(GOL),
        - 처소(LOC), 자극(STM), 원인(CAU), 비교기준(CRT), 시간(TMP), 정도(DGR), 방법(MNR),
        - 자격(ROL), 재료(MAT), 도구(INS), 경로(ROU), 방향(DIR), 수혜자(BEN), 내용(춧), 목적(PUR)

      o UPropBank: 표준국어대사전의 90,100개의 다의어 용언의 필수논항에 의미역 부착


    ◆ 구축 방법

    o UTagger-DP(반자동)를 이용하여 의존관계 부착
    o UTagger-SR(반자동, 통계기반)을 이용하여 의미역 부착

    ◆ 데이터 정보

     데이터 구조

      -BI tag


    ◆ 데이터 설명

    □ 개요
      o 약 2,000,000 어절의 형태소-동형이의어-의존관계-의미역 부착 말뭉치를 대상으로 개체명 부착
